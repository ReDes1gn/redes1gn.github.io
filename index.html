How it works

One screenshot to a full wall, in four steps.

01

Drop in a screenshot

One image, or a folder of the same screen.

02

Pick models + presets

Pick models and presets, or write your own. Add a style reference to steer the look.

03

Hit run

Calls fan out to every provider at once, with self-healing key rotation.

04

Browse the wall

Results land in a filterable grid, original shown first.

Built for real work

A serious tool under the pretty screenshots.

Every reimagining is a real, self-contained prototype, not a mockup.

Side-by-side viewer

Filter by model or preset; set columns and device width, desktop to phone.

Reference images

Drop in a style shot; every job borrows its mood and palette, not its layout.

Self-healing keys

Many keys per provider, round-robined. Flaky ones cool down and heal out of rotation.

Sandboxed rendering

Model-authored HTML and JS run in a sandboxed iframe. Interactive, but it cannot touch your data.

Runs locally

Pure Bun, localhost-only. Your screenshots go nowhere but the model APIs you pick.

Reproducible + scriptable

Every run is saved to a manifest. Full CLI, plus an MCP server for agents.

Even text-only models see the UI

Text-only models get an auto-caption first, so they reimagine the real screen.
