How it works

From one screenshot to a wall of redesigns, in four steps.

01

Drop in a screenshot

A single image, or a folder of shots of the same screen.

02

Pick models + presets

Choose your models and presets, or write your own prompt. Add a style reference if you want a particular look.

03

Hit run

Every model runs at the same time. If a key gets rate-limited, it quietly rotates to the next one.

04

Browse the wall

Everything shows up in a grid you can filter, with the original first so it is easy to compare.

Models

Every major model, plus any you add yourself.

Comes set up with Claude, GPT, Gemini, DeepSeek and Qwen. Point it at newer versions or add your own whenever you want.

Eight presets to start from, or write your own.

Built for real work

A real tool under the pretty screenshots.

Every result is a real, working page you can click around in, not a flat mockup.

Side-by-side viewer

Filter by model or preset, change the number of columns, and preview at desktop, laptop, tablet or phone width.

Reference images

Drop in a look you like and every model borrows its mood and colors, not its layout.

Self-healing keys

Give each provider a stack of API keys and it rotates through them. Ones that start failing get benched for a while, then come back on their own.

Sandboxed rendering

Whatever the models write runs in a locked-down iframe. You can click around in it, but it cannot reach your data or the rest of the app.

Runs locally

It runs on your own machine and only talks to localhost. Your screenshots never leave except to the model APIs you chose.

Reproducible + scriptable

Every run is written down so you can replay it later. There is a full command-line version too, plus an MCP server so agents can drive it.

Even text-only models see the UI

Models that cannot see images get a written description of your screenshot first, so they still redesign the real thing.

The wall, for real

One screenshot in. A dozen real redesigns out.

Straight from the viewer, on a real run of the dashboard above. Every result sits next to the original, tagged by model, preset, and what it cost.

Under the hood

Fast to run, safe to trust.

Everything at once

Every model runs at the same time, so a batch is only as slow as your slowest provider.

Localhost-only server

Built on Bun, and nothing listens outside your own machine.

Command line and agents

Anything you can do in the UI you can do from the command line, or hand off to an agent over MCP.

Manifests, not mystery

Every run is written down, so a good result is repeatable, not a fluke.
